llm.hostbun.cc

One OpenAI-compatible endpoint, two lanes — cloud (Crazyrouter, 300+ models) and local (this machine's GPU). Switch by changing the URL path. No API key needed; auth is handled server-side.

LaneBase URLBackendAuth
cloudhttps://llm.hostbun.cc/v1Crazyrouter → 300+ models (Claude, GPT, Gemini, DeepSeek, Qwen, Doubao, Grok…)key injected (none from you)
localhttps://llm.hostbun.cc/local/v1Local LM Studio on the pbox machine (RTX 4090) → google/gemma-4-26b-a4bnone

Switching cloud ↔ local

Identical OpenAI API on both. The only difference is the base URL path: use /v1 for cloud, /local/v1 to talk to the local machine. Same request/response shape.

# cloud (any of 247 models)
curl https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"claude-sonnet-4-6","messages":[{"role":"user","content":"hi"}]}'

# local machine (gemma on the GPU)
curl https://llm.hostbun.cc/local/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"google/gemma-4-26b-a4b","messages":[{"role":"user","content":"hi"}]}'

OpenAI SDK

from openai import OpenAI
# api_key can be any placeholder — the gateway injects the real one
cloud = OpenAI(base_url="https://llm.hostbun.cc/v1",       api_key="x")
local = OpenAI(base_url="https://llm.hostbun.cc/local/v1", api_key="x")

cloud.chat.completions.create(model="gpt-4o", messages=[{"role":"user","content":"hi"}])
local.chat.completions.create(model="google/gemma-4-26b-a4b", messages=[{"role":"user","content":"hi"}])

Cloud lane — Crazyrouter API /v1

OpenAI-compatible aggregator (upstream api.crazyrouter.com/v1). Everything under https://llm.hostbun.cc/ that isn't /local or /docs is proxied here with the key injected.

MethodPathPurpose
POST/v1/chat/completionsChat (streaming, tool-calling, structured outputs, vision)
POST/v1/completionsLegacy text completion
POST/v1/embeddingsEmbeddings (e.g. text-embedding-3-small)
POST/v1/images/generationsImage generation (DALL·E, Doubao Seedream, cogview…)
POST/v1/audio/speech · /v1/audio/transcriptionsTTS / STT
POST/v1/rerankReranking
GET/v1/modelsList all available models

Capabilities: chat completions, streaming ("stream": true), tool/function calling, structured outputs, vision, image / video / audio generation, embeddings, rerank, multimodal. Full reference: crazyrouter.com/docs · pricing: crazyrouter.com/pricing.

Streaming example

curl -N https://llm.hostbun.cc/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"gemini-2.5-pro","stream":true,"messages":[{"role":"user","content":"count to 5"}]}'

Local lane — talk to the machine /local/v1

Routes to LM Studio on the pbox box (RTX 4090) via llm.bofrid.dev. Private (data stays on-prem), free, fast. One pinned model:

ModelNotes
google/gemma-4-26b-a4b26B MoE · vision + tool-use · ~8K context · the only loaded model (requests for other ids are served by it)
# list local models
curl https://llm.hostbun.cc/local/v1/models

# chat with the local machine
curl https://llm.hostbun.cc/local/v1/chat/completions -H "Content-Type: application/json" \
  -d '{"model":"google/gemma-4-26b-a4b","messages":[{"role":"user","content":"describe this"}]}'

Vision works (send image_url content parts, incl. base64 data URLs). Anthropic-style /local/v1/messages and native /local/api/v1/chat are also reachable on the local box.
